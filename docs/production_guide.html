1. Project Overview

NxtHelp is a university help marketplace built with Django 4.2 comprising 3 apps:

App Purpose Key Features
accounts Authentication & profiles Register, login, profile mgmt, UPI/phone storage
work Core business logic Help requests, applications, Razorpay payments, notifications
chat Real-time messaging Chat between poster & helper via AJAX polling

Data Models

Model Description
Profile User extension — profession, skills, year, department, UPI ID, phone
Category Help request categories with icons and colors
HelpRequest Core entity — title, description, budget, deadline, urgency, status
Application Users apply to help — message, proposed budget, status tracking
Notification In-app notifications for 8 event types
Payment Razorpay transactions — order_id, payment_id, signature verification
ChatMessage Messages between poster and selected helper

Project Structure

nxthelp/
├── accounts/          # User auth & profiles
├── work/              # Help requests, applications, payments
├── chat/              # Messaging system
├── templates/         # Project-level templates (base.html)
├── static/css/        # Global stylesheets
├── nxthelp/           # Project config (settings.py, urls.py)
├── requirements.txt   # Dependencies
└── db.sqlite3         # Development database (not for production!)

2. Pre-Deployment Checklist

⚠️ CRITICAL Complete ALL items below before deploying to production.
  • Update requirements.txt with all production dependencies
  • Set DEBUG = False
  • Generate a new SECRET_KEY (never reuse the dev key)
  • Configure ALLOWED_HOSTS with your domain name
  • Switch database from SQLite to PostgreSQL
  • Set up Razorpay LIVE API keys (with KYC verified)
  • Configure static file serving (WhiteNoise or Nginx)
  • Enable HTTPS / SSL certificate
  • Run python manage.py check --deploy
  • Run python manage.py collectstatic
  • Set up error logging (Sentry recommended)
  • Configure email backend for password resets (optional)
  • Test all payment flows in Razorpay test mode first

3. Environment Variables (.env Setup)

Create a .env file in your project root. Never commit this file to Git.

# ─── Django Core ─────────────────────────────────────
DJANGO_SECRET_KEY=your-super-secret-random-key-50chars-minimum
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com

# ─── Database (PostgreSQL) ───────────────────────────
DB_ENGINE=django.db.backends.postgresql
DB_NAME=nxthelp_db
DB_USER=nxthelp_user
DB_PASSWORD=your-strong-db-password
DB_HOST=localhost
DB_PORT=5432

# ─── Razorpay (LIVE keys for production) ─────────────
RAZORPAY_KEY_ID=rzp_live_XXXXXXXXXXXXXXX
RAZORPAY_KEY_SECRET=XXXXXXXXXXXXXXXXXXXXXXXX

# ─── Email (optional, for password resets) ───────────
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password

Generate a Secure SECRET_KEY

python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
💡 Tip Add .env to your .gitignore file to prevent accidentally committing secrets.

4. Database Migration (SQLite → PostgreSQL)

Step 1: Install PostgreSQL Driver

pip install psycopg2-binary

Step 2: Create PostgreSQL Database

-- Connect to PostgreSQL as superuser
CREATE DATABASE nxthelp_db;
CREATE USER nxthelp_user WITH PASSWORD 'your-strong-password';
ALTER ROLE nxthelp_user SET client_encoding TO 'utf8';
ALTER ROLE nxthelp_user SET default_transaction_isolation TO 'read committed';
ALTER ROLE nxthelp_user SET timezone TO 'Asia/Kolkata';
GRANT ALL PRIVILEGES ON DATABASE nxthelp_db TO nxthelp_user;

Step 3: Export Existing Data (Optional)

# Export data from SQLite
python manage.py dumpdata --natural-foreign --natural-primary \
    -e contenttypes -e auth.Permission --indent 2 > datadump.json

Step 4: Migrate & Import

# After setting DB env vars to PostgreSQL
python manage.py migrate
python manage.py loaddata datadump.json   # Import old data
python manage.py createsuperuser          # Create admin user

5. Security Hardening

Add these settings to settings.py for production:

# ─── Production Security Settings ────────────────────
if not DEBUG:
    # HTTPS
    SECURE_SSL_REDIRECT = True
    SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
    
    # HSTS (HTTP Strict Transport Security)
    SECURE_HSTS_SECONDS = 31536000       # 1 year
    SECURE_HSTS_INCLUDE_SUBDOMAINS = True
    SECURE_HSTS_PRELOAD = True
    
    # Other Security
    SECURE_CONTENT_TYPE_NOSNIFF = True
    SECURE_BROWSER_XSS_FILTER = True
    X_FRAME_OPTIONS = 'DENY'
    CSRF_TRUSTED_ORIGINS = ['https://yourdomain.com']
🔒 Never Hardcode Secrets Always use environment variables for API keys, database passwords, and the Django secret key in production.

7. Razorpay: Test → Live Keys

Steps to Get Live Keys

  1. Go to dashboard.razorpay.com
  2. Complete KYC verification (PAN, bank account, business details)
  3. Switch to Live Mode (toggle at top of dashboard)
  4. Go to Settings → API Keys → Generate Live Key
  5. Copy Key ID (rzp_live_...) and Key Secret
  6. Set them as environment variables in your .env

Test Scenarios (Run Before Going Live)

Test Scenario Expected Result
Create order with valid amount Razorpay order created, checkout opens
Complete payment with test card 4111 1111 1111 1111 Payment verified, receipt generated
UPI test payment with success@razorpay Payment succeeds, status updated
Payment with insufficient funds Graceful error message shown to user
Close checkout modal mid-payment Button re-enables, no orphan records
Verify payment signature Signature verification passes correctly
⚠️ Important Always test your ENTIRE payment flow in Razorpay test mode before switching to live keys. Test keys start with rzp_test_, live keys start with rzp_live_.

Render Quick Deploy

  1. Push your code to GitHub
  2. Create a new Web Service on render.com
  3. Set all environment variables from Section 3
  4. Build command: pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate
  5. Start command: gunicorn nxthelp.wsgi:application

9. Nginx + Gunicorn Setup (VPS)

Install Dependencies

sudo apt update && sudo apt install python3-pip python3-venv nginx postgresql

Gunicorn Systemd Service

Create /etc/systemd/system/nxthelp.service:

[Unit]
Description=NxtHelp Gunicorn Daemon
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/home/deploy/nxthelp
EnvironmentFile=/home/deploy/nxthelp/.env
ExecStart=/home/deploy/nxthelp/venv/bin/gunicorn \
    --workers 3 \
    --bind unix:/run/nxthelp.sock \
    nxthelp.wsgi:application

[Install]
WantedBy=multi-user.target

Nginx Site Configuration

Create /etc/nginx/sites-available/nxthelp:

server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location /static/ {
        alias /home/deploy/nxthelp/staticfiles/;
    }

    location / {
        proxy_pass http://unix:/run/nxthelp.sock;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Enable & Start

sudo ln -s /etc/nginx/sites-available/nxthelp /etc/nginx/sites-enabled
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nxthelp && sudo systemctl start nxthelp

Add Redis Caching

# settings.py
CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.redis.RedisCache',
        'LOCATION': 'redis://127.0.0.1:6379/1',
    }
}

Query Optimization Tips

  • Your views already use select_related() — this is excellent!
  • Add database indexes on frequently filtered fields (status, created_at)
  • Use prefetch_related() for reverse FK lookups on list pages
  • Use Django's cached_db session backend for faster sessions

13. Backup Strategy

Daily PostgreSQL Backups

# Manual backup
pg_dump -U nxthelp_user nxthelp_db > backup_$(date +%Y%m%d).sql

# Automated (add to crontab — runs daily at 2 AM)
0 2 * * * pg_dump -U nxthelp_user nxthelp_db | gzip > /backups/nxthelp_$(date +\%Y\%m\%d).sql.gz

# Restore from backup
psql -U nxthelp_user nxthelp_db < backup_20260301.sql
